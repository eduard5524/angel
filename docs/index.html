Angel Documentation

Angel is an open-source tool for self-hosting large language models. It gives you a private, OpenAI-compatible inference endpoint running on your own infrastructure — with zero API costs.

New to Angel? Follow the Quick Start guide to get your first model running in under 5 minutes.

Installation

Prerequisites

  • Python 3.10 or higher
  • NVIDIA GPU with CUDA 12.1+ (compute capability 7.0+)
  • At least 16 GB of RAM (model-dependent)
  • Linux (Ubuntu 22.04+ recommended) or WSL2 on Windows

Install via pip

pip install angel-ai

Install from source

git clone https://github.com/eduard5524/angel.git
cd angel
pip install -e ".[dev]"

Verify installation

angel --version
# Angel v1.0.0

angel doctor
# Checking CUDA... OK (CUDA 12.4)
# Checking GPU... OK (NVIDIA A100 80GB)
# Checking disk space... OK (234 GB free)
# All checks passed!

Quick Start

Get a model running in three steps:

1. Pull a model

# Pull a model (downloads and optimizes automatically)
angel pull deepseek-r1:7b

# List available models
angel list --remote

# See downloaded models
angel list

2. Start the server

# Start serving on port 8000
angel serve deepseek-r1:7b --port 8000

# With custom options
angel serve deepseek-r1:7b \
    --port 8000 \
    --host 0.0.0.0 \
    --max-concurrent 8 \
    --gpu-memory-utilization 0.9

3. Send a request

# Using curl
curl http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "deepseek-r1:7b",
    "messages": [
      {"role": "user", "content": "Hello, how are you?"}
    ]
  }'
# Using Python
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:8000/v1",
    api_key="not-needed"
)

response = client.chat.completions.create(
    model="deepseek-r1:7b",
    messages=[
        {"role": "user", "content": "Explain quantum computing."}
    ]
)
print(response.choices[0].message.content)

Configuration

Angel can be configured via a YAML file, environment variables, or command-line flags.

Configuration file

Create angel.yaml in your project root or ~/.angel/config.yaml:

# angel.yaml
server:
  host: 0.0.0.0
  port: 8000
  workers: 1
  cors_origins: ["*"]

model:
  name: deepseek-r1:7b
  quantization: auto        # auto, awq, gptq, none
  gpu_memory_utilization: 0.9
  max_model_len: 8192
  trust_remote_code: true

inference:
  max_concurrent_requests: 16
  timeout: 120
  temperature: 0.7
  max_tokens: 4096

logging:
  level: info               # debug, info, warning, error
  format: json
  file: /var/log/angel/server.log

Environment variables

All configuration options can be set via environment variables prefixed with ANGEL_:

export ANGEL_SERVER_PORT=8000
export ANGEL_SERVER_HOST=0.0.0.0
export ANGEL_MODEL_NAME=deepseek-r1:7b
export ANGEL_MODEL_GPU_MEMORY_UTILIZATION=0.9
export ANGEL_INFERENCE_MAX_CONCURRENT_REQUESTS=16

CLI flags reference

Flag Description Default
--port Server port 8000
--host Bind address 127.0.0.1
--gpu-memory-utilization Fraction of GPU memory to use 0.9
--max-model-len Maximum sequence length Model default
--quantization Quantization method (auto, awq, gptq, none) auto
--max-concurrent Max concurrent requests 16
--tensor-parallel Number of GPUs for tensor parallelism 1
--log-level Logging level info

Supported Models

Angel supports all major open-source language models. Each model is optimized for fast inference with automatic quantization.

DeepSeek

DeepSeek models excel at reasoning, mathematics, and code generation. The R1 series uses chain-of-thought reasoning for complex problem solving.

Model Parameters VRAM Required Context Length
deepseek-r1:7b 7B ~6 GB 128K
deepseek-r1:32b 32B ~24 GB 128K
deepseek-r1:70b 70B ~48 GB 128K

# Pull and run DeepSeek
angel pull deepseek-r1:7b
angel serve deepseek-r1:7b --port 8000

# For larger variants, use multi-GPU
angel serve deepseek-r1:70b --tensor-parallel 2 --port 8000

Infrastructure Setup

Angel runs on any machine with an NVIDIA GPU. Below are guides for the most common deployment targets.

Minimum requirements: NVIDIA GPU with CUDA 12.1+, 16 GB RAM, 50 GB disk space. Actual requirements depend on the model size.

Azure GPU VMs

Azure offers NC, ND, and NV-series VMs with NVIDIA GPUs. Recommended for enterprise deployments with Azure's global network and compliance certifications.

Recommended VM sizes

VM Size GPU VRAM Best For
Standard_NC4as_T4_v3 1x T4 16 GB 7B models
Standard_NC24ads_A100_v4 1x A100 80 GB Up to 70B models
Standard_ND96asr_v4 8x A100 640 GB 405B+ models
Standard_ND96isr_H100_v5 8x H100 640 GB Max performance

Setup steps

# 1. Create the VM (using Azure CLI)
az vm create \
  --resource-group myResourceGroup \
  --name angel-server \
  --image Canonical:0001-com-ubuntu-server-jammy:22_04-lts-gen2:latest \
  --size Standard_NC24ads_A100_v4 \
  --admin-username azureuser \
  --generate-ssh-keys

# 2. SSH into the VM
ssh azureuser@<public-ip>

# 3. Install NVIDIA drivers and CUDA
sudo apt update && sudo apt install -y nvidia-driver-535 nvidia-cuda-toolkit

# 4. Install Angel
pip install angel-ai

# 5. Pull and serve a model
angel pull deepseek-r1:7b
angel serve deepseek-r1:7b --host 0.0.0.0 --port 8000

Important: Make sure to open port 8000 in your Network Security Group (NSG) rules, or use a reverse proxy with HTTPS for production.

DigitalOcean GPU Droplets

DigitalOcean's GPU Droplets offer simple, flat-rate pricing with H100 GPUs. Great for developers who want a straightforward setup.

Available GPU Droplets

Droplet Type GPU VRAM Price
gpu-h100x1-80gb 1x H100 80 GB ~$2.50/hr
gpu-h100x8-640gb 8x H100 640 GB ~$20/hr

Setup steps

# 1. Create a GPU Droplet via doctl
doctl compute droplet create angel-server \
  --size gpu-h100x1-80gb \
  --image ubuntu-22-04-x64 \
  --region nyc1 \
  --ssh-keys <your-key-id>

# 2. SSH into the Droplet
ssh root@<droplet-ip>

# 3. NVIDIA drivers are pre-installed on GPU Droplets
nvidia-smi  # Verify GPU is detected

# 4. Install Angel
pip install angel-ai

# 5. Pull and serve
angel pull llama3.3:70b
angel serve llama3.3:70b --host 0.0.0.0 --port 8000

Running as a systemd service

# Create service file: /etc/systemd/system/angel.service
[Unit]
Description=Angel AI Model Server
After=network.target

[Service]
Type=simple
User=angel
ExecStart=/usr/local/bin/angel serve deepseek-r1:7b --host 0.0.0.0 --port 8000
Restart=always
RestartSec=10
Environment=ANGEL_MODEL_GPU_MEMORY_UTILIZATION=0.9

[Install]
WantedBy=multi-user.target
# Enable and start
sudo systemctl daemon-reload
sudo systemctl enable angel
sudo systemctl start angel

# Check status
sudo systemctl status angel
journalctl -u angel -f

Multi-GPU Setup

Angel supports tensor parallelism for distributing large models across multiple GPUs. This is essential for running models that don't fit in a single GPU's memory.

Tensor parallelism

# Automatically distribute across all available GPUs
angel serve deepseek-r1:70b --tensor-parallel auto

# Specify exact number of GPUs
angel serve deepseek-r1:70b --tensor-parallel 2

# Choose specific GPU devices
CUDA_VISIBLE_DEVICES=0,1 angel serve deepseek-r1:70b --tensor-parallel 2

GPU requirements by model size

Model 1x A100 80GB 2x A100 80GB 4x A100 80GB 8x A100 80GB
7B–8B Yes Yes Yes Yes
32B–34B Yes Yes Yes Yes
70B–72B Tight Yes Yes Yes
405B No No Yes (AWQ) Yes
671B (MoE) No No No Yes

API Reference

Angel exposes an OpenAI-compatible REST API. Any client library or tool that works with OpenAI's API will work with Angel.

Chat Completions

POST /v1/chat/completions

curl http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "deepseek-r1:7b",
    "messages": [
      {"role": "system", "content": "You are a helpful assistant."},
      {"role": "user", "content": "What is machine learning?"}
    ],
    "temperature": 0.7,
    "max_tokens": 1024,
    "stream": false
  }'

Request parameters

Parameter Type Description
model string Model identifier (required)
messages array Array of message objects (required)
temperature float Sampling temperature (0–2, default: 0.7)
max_tokens integer Maximum tokens to generate
stream boolean Stream responses via SSE (default: false)
top_p float Nucleus sampling parameter (default: 1.0)
stop string/array Stop sequences

Streaming responses

# Stream with curl
curl http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "deepseek-r1:7b",
    "messages": [{"role": "user", "content": "Write a poem."}],
    "stream": true
  }'

# Stream with Python
from openai import OpenAI

client = OpenAI(base_url="http://localhost:8000/v1", api_key="not-needed")

stream = client.chat.completions.create(
    model="deepseek-r1:7b",
    messages=[{"role": "user", "content": "Write a poem."}],
    stream=True
)

for chunk in stream:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="")

Quantization

Quantization reduces model size and memory usage, allowing larger models to run on less hardware. Angel supports several quantization methods.

Method Compression Quality Impact Speed
FP16 (none) 1x (baseline) None Fast
AWQ (4-bit) ~4x Minimal Fastest
GPTQ (4-bit) ~4x Minimal Fast
GGUF (variable) 2–8x Varies Good 
# Serve with specific quantization
angel serve deepseek-r1:70b --quantization awq

# Auto-detect best quantization for your hardware
angel serve deepseek-r1:70b --quantization auto

Performance Tuning

Tips for maximizing throughput and minimizing latency.

GPU memory utilization

# Use 95% of GPU memory (leaves 5% for overhead)
angel serve deepseek-r1:7b --gpu-memory-utilization 0.95

Batch size optimization

# Increase max concurrent requests for throughput
angel serve deepseek-r1:7b --max-concurrent 32

Context length

# Limit context length to save memory
angel serve deepseek-r1:7b --max-model-len 4096

Benchmarking

# Run built-in benchmarks
angel benchmark deepseek-r1:7b \
    --num-prompts 100 \
    --concurrent 8 \
    --prompt-length 256 \
    --output-length 512

Troubleshooting

Common issues

CUDA out of memory

# Solution 1: Lower GPU memory utilization
angel serve model --gpu-memory-utilization 0.8

# Solution 2: Use a smaller context length
angel serve model --max-model-len 2048

# Solution 3: Use quantization
angel serve model --quantization awq

# Solution 4: Use tensor parallelism (multi-GPU)
angel serve model --tensor-parallel 2

Slow inference

# Check GPU utilization
nvidia-smi

# Ensure CUDA is properly installed
angel doctor

# Try AWQ quantization for faster inference
angel serve model --quantization awq

Model download fails

# Retry with verbose logging
angel pull model --log-level debug

# Use a mirror
angel pull model --mirror https://mirror.example.com

# Check disk space
df -h

Port already in use

# Use a different port
angel serve model --port 8001

# Find and kill the process using port 8000
lsof -i :8000
kill <PID>
